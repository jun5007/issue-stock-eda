## 데이터 처리 및 피처 엔지니어링

### 전처리 흐름

1. 종목별 Wide Format 데이터를 `date`, `종목`, `변수` 기준 Long Format으로 변환
2. 뉴스/검색 데이터와 주식 데이터를 `date`, `종목` 기준으로 병합
3. 휴장일 이슈를 다음 거래일 반응과 연결하기 위해 `next_trade_date` 생성
4. 가격/보유량 계열은 Forward Fill 적용
5. 거래량/수급 같은 흐름 변수는 결측 시 0으로 처리
6. 뉴스 건수와 검색량에 로그 변환 적용

### 주요 파생 피처

| 구분 | 변수 | 설명 |
|---|---|---|
| 뉴스 | `뉴스_log` | `log1p(뉴스건수)` |
| 검색 | `검색_log` | `log1p(검색량)` |
| 이슈 | `이슈강도` | `뉴스_log + 검색_log` |
| 변동성 | `등락률_abs` | 등락률 절댓값 |
| 거래량 | `거래량_log` | `log1p(거래량)` |
| 선행 반응 | `등락률_next` | 다음 거래일 등락률 |
| 선행 반응 | `거래량_log_next` | 다음 거래일 로그 거래량 |

## 분석 구조

### 1. 이슈 민감도 기반 종목 클러스터링

| 항목 | 내용 |
|---|---|
| 입력 변수 | 수익률 변동성, 거래량 스파이크 |
| 전처리 | StandardScaler |
| 모델 | K-Means |
| 선택 군집 수 | K=3 |
| 판단 기준 | Silhouette Score와 분석 목적을 함께 고려 |

### 2. 이벤트 스터디

- 종목별 이슈강도 Top1 날짜를 이벤트일로 정의
- 이벤트 전후 수익률과 거래량 변화를 비교
- 휴장일에 발생한 이슈는 다음 거래일 반응과 연결

### 3. 상관관계 분석

- 이슈강도와 다음 거래일 등락률의 관계 분석
- 이슈강도와 다음 거래일 거래량의 관계 분석
- 종목별 상관계수 차이 비교

### 주요 인사이트

- 이슈강도와 다음 거래일 등락률의 전체 상관계수는 약 `0.03`으로 매우 낮았습니다.
- 이슈강도와 다음 거래일 거래량의 상관계수는 약 `0.56`으로 더 뚜렷했습니다.
- 전 종목에서 이슈강도와 다음날 거래량 변화의 상관계수가 양수로 나타났습니다.
- 이슈강도는 가격 방향을 직접 예측하는 변수라기보다 투자자 관심도와 거래 참여 강도를 설명하는 지표에 더 가깝습니다.
